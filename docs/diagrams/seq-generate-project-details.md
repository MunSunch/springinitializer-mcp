# Генерация проекта — подробное описание шагов

Подробное описание каждого нумерованного шага из `seq-generate-project.puml`.

## Шаг 1 — Пользователь формулирует запрос
Пользователь вводит запрос на естественном языке, например: «Создай Spring Boot проект с Web и JPA». AI-ассистент интерпретирует запрос и решает использовать MCP-сервер.

## Шаг 2 — Клиент читает ресурс с опциями
AI-ассистент вызывает `resources/read` с URI `springinitializr://options`, чтобы узнать доступные типы проектов, языки, версии Java, типы упаковки и версии Spring Boot.

## Шаг 3 — Сервер запрашивает метаданные у API
Сервер отправляет HTTPS-запрос к `https://start.spring.io/metadata/client` для получения актуальных метаданных. Это первый вызов, поэтому кэш пуст.

## Шаг 4 — API возвращает метаданные
Spring Initializr возвращает JSON-объект со всеми доступными опциями конфигурации: поддерживаемые версии Spring Boot, версии Java, типы проектов и категории зависимостей.

## Шаг 5 — Сервер возвращает отформатированные опции
Сервер форматирует метаданные в читаемый текст и возвращает его AI-ассистенту.

## Шаг 6 — Клиент читает ресурс с зависимостями
AI-ассистент вызывает `resources/read` с URI `springinitializr://dependencies`, чтобы увидеть все доступные зависимости по категориям.

## Шаг 7 — Сервер возвращает зависимости (из кэша)
Сервер использует кэшированные метаданные из шага 3 и возвращает отформатированные категории зависимостей (Web, SQL, Security, AI и др.) со всеми идентификаторами.

## Шаг 8 — AI выбирает параметры
На основе запроса пользователя и доступных опций/зависимостей AI-ассистент подбирает параметры: тип проекта (gradle-project), язык (java), зависимости (web, data-jpa) и т.д.

## Шаг 9 — Клиент вызывает инструмент генерации
AI-ассистент отправляет запрос `tools/call` для инструмента `generate-spring-boot-project` с выбранными параметрами.

## Шаг 10 — Сервер передаёт вызов обработчику
MCP-сервер маршрутизирует вызов инструмента к зарегистрированному обработчику в `generate-project.ts`.

## Шаг 11 — Формирование URL загрузки
Инструмент вызывает `buildDownloadUrl()` с параметрами запроса (имена ключей маппятся из параметров инструмента в имена API, например `projectType` → `type`) для построения URL загрузки.

## Шаг 12 — Кодирование параметров
Функция перебирает все параметры, кодирует значения через `encodeURIComponent` и соединяет их в строку запроса, добавляемую к `https://start.spring.io/starter.zip`.

## Шаг 13 — Возврат URL загрузки
Сформированный URL (например, `https://start.spring.io/starter.zip?type=gradle-project&language=java&dependencies=web%2Cdata-jpa`) возвращается инструменту.

## Шаг 14 — Вызов fetchZip
Инструмент вызывает `fetchZip()` с URL загрузки для получения ZIP-архива.

## Шаг 15 — Загрузка ZIP-архива проекта
Утилита выполняет HTTPS GET-запрос к Spring Initializr API с построенным URL. Ответ — ZIP-файл со сгенерированным Spring Boot проектом.

## Шаг 16 — API отдаёт ZIP-поток
Spring Initializr генерирует проект на лету и отдаёт ZIP-файл. Ответ загружается в память как `Buffer`.

## Шаг 17 — Возврат Buffer инструменту
Функция загрузки возвращает `Buffer` с содержимым ZIP-архива обработчику инструмента.

## Шаг 18 — Инструмент формирует результат
Инструмент кодирует ZIP-архив в base64, оборачивает в MCP-ресурс с типом `application/zip` и добавляет текстовое сообщение с именем файла и размером.

## Шаг 19 — Сервер отправляет ответ клиенту
MCP-сервер оборачивает результат в JSON-RPC ответ и отправляет через транспорт AI-ассистенту.

## Шаг 20 — AI представляет результат пользователю
AI-ассистент формирует понятный ответ: где проект, какие зависимости включены, размер архива.
