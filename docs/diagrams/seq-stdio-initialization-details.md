# Инициализация через stdio — подробное описание шагов

Подробное описание каждого нумерованного шага из `seq-stdio-initialization.puml`.

## Шаг 1 — Пользователь запускает AI-ассистента
Пользователь запускает AI-ассистента (Claude Desktop, Cursor и др.), в конфигурации которого указан данный MCP-сервер. Ассистент читает команду запуска MCP-сервера из файла конфигурации.

## Шаг 2 — AI-ассистент порождает процесс MCP-сервера
MCP-клиент запускает сервер как дочерний процесс командой из конфигурации (например, `node dist/index.js`). Обмен данными происходит через stdin/stdout.

## Шаг 3 — Разбор аргументов CLI
Сервер считывает `process.argv` для определения режима транспорта. Без флага `--transport http` используется режим `stdio` по умолчанию.

## Шаг 4 — Создание экземпляра McpServer
Создаётся объект `McpServer` с именем сервера (`springinitializr`) и версией (`1.0.0`). Это основной обработчик MCP-протокола.

## Шаг 5 — Регистрация инструментов, ресурсов, промптов
Вызываются три функции регистрации:
- `registerTools()` — регистрирует инструмент `generate-spring-boot-project`
- `registerResources()` — регистрирует ресурсы с метаданными (опции, зависимости)
- `registerPrompts()` — регистрирует промпт для генерации проекта

## Шаг 6 — Создание StdioServerTransport
Создаётся экземпляр `StdioServerTransport`. Этот транспорт читает JSON-RPC сообщения из `process.stdin` и пишет ответы в `process.stdout`.

## Шаг 7 — Подключение сервера к транспорту
`server.connect(transport)` привязывает MCP-сервер к stdio-транспорту. Транспорт начинает слушать входящие сообщения на stdin.

## Шаг 8 — Транспорт слушает stdin/stdout
Транспорт настраивает обработчики потоков на `process.stdin` для приёма JSON-RPC сообщений и использует `process.stdout` для отправки ответов. Логирование идёт в stderr, чтобы не мешать протоколу.

## Шаг 9 — MCP-клиент отправляет запрос initialize
AI-ассистент отправляет JSON-RPC запрос `initialize` с версией MCP-протокола, возможностями клиента и информацией о клиенте.

## Шаг 10 — Транспорт маршрутизирует к обработчику
Stdio-транспорт парсит JSON-RPC сообщение и направляет метод `initialize` внутреннему обработчику MCP-сервера.

## Шаг 11 — Сервер отправляет ответ initialize
Сервер отвечает со своими возможностями (инструменты, ресурсы, промпты), версией протокола и информацией о сервере.

## Шаг 12 — Транспорт доставляет ответ
Ответ сериализуется в JSON-RPC и записывается в stdout для чтения MCP-клиентом.

## Шаг 13 — Клиент отправляет уведомление initialized
Клиент отправляет уведомление `notifications/initialized` для подтверждения завершения рукопожатия.

## Шаг 14 — Сервер обрабатывает уведомление
Сервер обрабатывает уведомление. Сессия полностью установлена, клиент может вызывать инструменты, читать ресурсы и использовать промпты.
